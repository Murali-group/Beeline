In order to process the output of BoolODE simulations, configure settings under post_processsing in the config file. Ensure that do_post_processing is set to True.

Generate Samples from Simulations¶

The default post processing option is genSamples. Even if genSamples is not specified by the user, as long as any other post processing methods are specified, at least one dataset is generated by genSamples for further processing. This ensures that the simulation files remain untouched, while working with datasets of maneagable sizes.

genSamples takes two further parameters

  1. sample_size specifies the number of cells (< num_cells) to sample from the simulations.
  2. nDatasets specifies the number of datasets with sample_size simulation time points, i.e. cells, to sample.

Note

If genSamples is not explicitly specified, genSamples() is still run with sample_size=num_cells and nDatasets=1

Carry out dimensionality reduction¶

Having sampled the simulated trajectories, we can now visualize the generated dataset, by carrying out dimensionality reduction using t-SNE. This is specified using DimRed which further takes a list of perplexities. tSNE is performed on each generated dataset with each specified perplexity. This can help in picking the appropriate value of perplexity for downstream analysis.

Remember, each cell is a time point from a simulation. Thus, the low dimensional representation of cells can be colored by their “simulation time” which is inferred from the cell ID. Darker colors imply early time points in the simulation. The output should look like the following

Generate dropouts from expression data¶

In order to mimic real single-cell expression datasets, BoolODE includes post_processing.genDropouts() which implements dropouts as described in the paper, by dropping expression values below drop_cutoff using a probability of drop_prob.

Attention

Ensure the - dropout: True is specified. If not, genDropouts will still randomly sample cells but not dropout any values.

Compute Pseudotime using Slingshot¶

Note

runSlingshot.py requires the SlingShot docker file! Please make sure docker has been setup and the container built.

Slingshot computes pseudotime trajectories for a given dataset by first carrying out dimensionality reduction, then carrying out k-means clustering on the low dimensional embedding in order to compute trajectories. The number of clusters expected depend on the features of the dataset. For instance, a dataset with two steady state clusters should be specified with nClusters: 3, specifying an additional initial state cluster.
